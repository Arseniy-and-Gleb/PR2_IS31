# Для чего нужен файл README.md в репозитории?
Файл README.md — это главный информационный файл в репозитории, который выполняет несколько важных функций. 
Во-первых, он служит визитной карточкой проекта, создавая первое впечатление у посетителей репозитория. 
Во-вторых, это основная документация, которая объясняет, что представляет собой проект, для чего он предназначен и какую проблему решает. 
В-третьих, README.md содержит практические инструкции по установке, настройке и использованию проекта, что особенно важно для новых пользователей и разработчиков
# Какую информацию обычно размещают в файле README.md?
В README.md обычно размещают структурированную информацию о проекте. Начинается файл с названия проекта и краткого описания его сути. 
Далее следует раздел с возможностями или особенностями, где перечислены ключевые функции. Обязательным элементом являются инструкции по установке и настройке, часто с примерами команд для терминала
# Как создать заголовок первого уровня в Markdown?
Чтобы создать заголовок первого уровня в Markdown, нужно поставить символ решетки # перед текстом заголовка. Например, запись # Название проекта создаст заголовок самого крупного размера
# Как создать нумерованный список в Markdown?
Для создания нумерованного списка в Markdown нужно начать каждую строку с числа, точки и пробела. При этом сами числа могут быть последовательными (1, 2, 3) или одинаковыми (1, 1, 1) — Markdown автоматически преобразует их в правильную нумерацию. 
Оба варианта дадут одинаковый результат. Для создания вложенных списков нужно добавить отступ в четыре пробела или одну табуляцию перед номером вложенного пункта.
Нумерованные списки можно комбинировать с маркированными внутри одного списка, используя соответствующие символы для каждого уровня вложенности. Это позволяет создавать сложные иерархические структуры документации
# Для чего нужна вики-страница в репозитории?
Вики-страница в репозитории GitHub служит расширенной системой документации, которая дополняет базовую информацию из README.md. 
Она предназначена для хранения подробной, многостраничной документации, которая не помещается в один файл или требует постоянного обновления. 
Вики особенно полезна для крупных проектов, где необходимо документировать API, архитектуру, процессы разработки, руководства пользователя и другую техническую информацию
